## 1. バージョンタグとコミットハッシュの違い

### バージョンタグ

例：v1
バージョンタグは可変（Mutable）である
タグは単なる付箋であり、内部のコードを変更した場合でもv1と名乗れることが可能である

### コミットハッシュ
統計

例：f2a4c...
コミットハッシュは不変（Immutable）である
SHAハッシュはコードの内容から生成される暗号
コード内容が変わればハッシュも変わる。つまり、特定のSHAを指定すれば、いつ実行しても100%同一のコードが実行されるということである

## 2. セキュリティリスクの例

### サプライチェーン攻撃（タグの書き換え）
人気のアクション（例: awesome-action）の作者のアカウントがハックされた場合
1. 攻撃者は（例: awesome-action）に悪意のあるコードを混入
2. 攻撃者はv1タグを削除し、悪意あるコードを含む新しいコミットv1タグを付け直す
3. 自身の設定ファイルはuses: owner/awesome-action@v1のまま
4. 次回の実行時に自動的に悪意あるコードがダウンロードされ実行されてしまう。

## 3. 具体的な記述例

### 非推奨
```
steps:
  - uses: actions/checkout@v3
```
リスク：v3が指す先はいつでも変更される可能性がある

### 推奨
```
steps:
  - uses: actions/checkout@f43a0e5ff2bd294095638e18286ca9a3d1956744 # v3.6.0
```
安全性：常にこのハッシュ時点でのコードしか実行されない
ポイント：verの確認がわからなくなるため、コメントでversionを記載しておくと可読性の向上
